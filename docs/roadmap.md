# Picko Roadmap

> 시장과 감정 맥락을 읽고, 브랜드 컨셉을 정의하고, 좋은 레퍼런스를 선별·평가한 뒤, 전략과 크리에이티브를 생성·실험·학습하는 **마케터 에이전틱 시스템**

**상세 스펙:** [`specs/015-marketing-os/spec.md`](../specs/015-marketing-os/spec.md)

---

## 시스템 본질

이 시스템은 단순한 콘텐츠 생성기가 아니다.

**브랜드 전략가 + 리서처 + 크리에이티브 디렉터 + 퍼포먼스 마케터**를 분해한 시스템이다.

```
┌─────────────────────────────────────────────────────────────┐
│                      브랜드 두뇌                             │
│  우리는 누구인가 / 어떻게 읽혀야 하는가 / 무엇으로 오해되면 안 되는가  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                       시장 눈                                │
│  지금 사람들은 어떤 감정 구조에 반응하는가 / 어떤 레퍼런스가 진짜 유효한가  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    크리에이티브 손                            │
│          그 판단을 바탕으로 실제 콘텐츠를 만든다                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      실험 뇌                                 │
│          무엇이 먹혔고 왜 먹혔는지 학습한다                     │
└─────────────────────────────────────────────────────────────┘
```

**핵심 원칙:** 생성 에이전트보다 먼저, 브랜드 개념을 만드는 에이전트와 좋은 데이터를 선별하는 에이전트가 상위에 있어야 한다.

---

## 6-Layer 아키텍처

### Layer 1. 시장/고객/경쟁 이해

| 에이전트 | 역할 |
|---------|------|
| 시장 탐색 에이전트 | 시장 트렌드, 카테고리 동향 파악 |
| 고객 감정 탐색 에이전트 | 타겟 오디언스의 감정 구조, 페인포인트 분석 |
| 경쟁 포지셔닝 에이전트 | 경쟁사 포지셔닝, 차별화 포인트 도출 |
| **데이터 스카우트 에이전트** | 좋은 학습 데이터를 선별하는 큐레이터 |

### Layer 2. 브랜드 정의

| 에이전트 | 역할 |
|---------|------|
| **브랜드 컨셉 에이전트** | 브랜드 운영체제를 만드는 핵심 에이전트 |
| 톤앤매너 에이전트 | 브랜드 표현 스타일 정의 |
| 금지 해석/리스크 에이전트 | 오해 방지, 위험 표현 식별 |
| 메시지 구조 에이전트 | 메시지 하우스, 핵심 메시지 구조화 |

### Layer 3. 레퍼런스 인텔리전스

| 에이전트 | 역할 |
|---------|------|
| 레퍼런스 수집 에이전트 | 레퍼런스 발굴 |
| 콘텐츠 원자 추출 에이전트 | 레퍼런스에서 패턴/요소 추출 |
| 댓글 해석 에이전트 | 반응 분석, 감정 해석 |
| 적합도 평가 에이전트 | 브랜드 컨셉 대비 레퍼런스 적합성 평가 |

### Layer 4. 전략 기획

| 에이전트 | 역할 |
|---------|------|
| 채널 전략 에이전트 | 채널별 접근 전략 |
| 캠페인 기획 에이전트 | 캠페인 구조, 네러티브 설계 |
| 시리즈 구조 에이전트 | 콘텐츠 시리즈 기획 |
| 실험 설계 에이전트 | A/B 테스트, 실험 설계 |

### Layer 5. 제작

| 에이전트 | 역할 |
|---------|------|
| 카피 생성 에이전트 | 텍스트 카피 생성 |
| 스토리보드 에이전트 | 영상/이미지 스토리보드 |
| 이미지 프롬프트 에이전트 | 이미지 생성 프롬프트 |
| 영상 프롬프트/스티치 에이전트 | 영상 생성, 편집 가이드 |
| 랜딩/블로그/Threads 변환 에이전트 | 포맷 변환 |

### Layer 6. 운영 최적화

| 에이전트 | 역할 |
|---------|------|
| 게시 캘린더 에이전트 | 게시 스케줄링 |
| 성과 해석 에이전트 | 성과 분석, 인사이트 도출 |
| 학습 반영 에이전트 | 성과 기반 학습, 개선점 도출 |
| 브랜드 진화 제안 에이전트 | 브랜드 컨셉 업데이트 제안 |

## 우선순위 (중요)

**"생성이 핵심"이 아니라 "판단이 핵심"이다.**

| 순위 | 에이전트 | 이유 |
|-----|---------|------|
| **1** | 브랜드 컨셉 에이전트 | 이게 없으면 레퍼런스 평가 기준도 흔들린다 |
| **2** | 데이터 스카우트 에이전트 | 나쁜 데이터를 많이 모으면 뒤 전부 오염된다 |
| **3** | 레퍼런스 평가 에이전트 | 좋은 것과 나쁜 것을 가르는 심판이 필요하다 |
| **4** | 전략/기획 에이전트 | 어떤 채널에 어떤 메시지 구조로 낼지 정해야 한다 |
| **5** | 생성 에이전트 | 맨 마지막이다. 생성은 상위 판단을 실행하는 손일 뿐이다 |

## 핵심 에이전트 상세

### 1. 브랜드 컨셉 에이전트 (1순위)

단순히 "예쁜 카피"를 만드는 게 아니다. **브랜드의 해석 기준을 고정**하는 역할이다.

**책임:**
- 고객 문제를 감정 언어로 재정의
- 경쟁 카테고리와의 차이 정의
- 브랜드의 핵심 감정축 정의
- 원하지 않는 해석 정의
- 톤앤매너 정의
- 금지 표현 정의
- 핵심 메시지 구조 정의
- 브랜드 세계관 초안 작성
- 채널별 표현 원칙 정의

**산출물:**
- Brand Concept Sheet
- Positioning Statement
- Emotional Territory Map
- Message House
- Tone & Manner Guide
- Misread Risk Guide

→ 이 에이전트는 생성기의 입력값이 되는 **브랜드 운영체제**를 만든다.

### 2. 데이터 스카우트 에이전트 (2순위)

많이 긁어오는 크롤러가 아니다. **"쓸 만한 데이터만 모으는 선별형 수집가"**여야 한다.

**책임:**
- 직접 경쟁사 탐색
- 인접 감정 카테고리 탐색
- 인접 미학 카테고리 탐색
- 인접 카피 패턴 탐색
- 광고 라이브러리 탐색
- 댓글 반응이 풍부한 사례 우선 수집
- 트렌드성 잡음/낚시성 데이터 제거
- 계정 규모/업로드 시점/포맷 차이를 정규화
- 데이터 소스 신뢰도 태깅

**산출물:**
- Reference Candidate Pool
- Source Credibility Score
- Noise/Contamination Flags
- Category Coverage Report

→ 이 에이전트는 단순 수집이 아니라 **좋은 학습재료를 골라오는 데이터 큐레이터**다.

## 필수 요구사항 체크리스트

### 브랜드 컨셉 파트
- [ ] 브랜드 컨셉을 정의하는 전용 에이전트 포함
- [ ] 포지셔닝, 감정 축, 톤앤매너, 금지 해석, 메시지 하우스 산출
- [ ] 생성 단계는 반드시 이 산출물을 참조

### 데이터 수집 파트
- [ ] 단순 크롤링이 아니라 선별하는 스카우트 에이전트 포함
- [ ] 직접 경쟁 + 감정 인접 + 미학 인접 + 카피 인접 카테고리 포함
- [ ] 오염 요인(유명인, 경품, 낚시, 논란성 버프) 식별 및 태깅

### 전략 파트
- [ ] 단건 콘텐츠 생성이 아니라 캠페인/시리즈/채널 운영 전략 설계
- [ ] 브랜드 개념과 레퍼런스 인텔리전스 기반 콘텐츠 기획안 도출

### 학습 파트
- [ ] 성과 좋은 콘텐츠 학습
- [ ] 브랜드 해석 실패 콘텐츠 역학습 → 금지 규칙 갱신
