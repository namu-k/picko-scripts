## 1. 시스템 개요

Picko는 콘텐츠 파이프라인 자동화 시스템으로, RSS 피드 및 다양한 웹 소스에서 콘텐츠를 자동 수집하고, AI를 활용해 다양한 형식의 콘텐츠를 생성한 후 게시까지 자동화합니다.

### 1.1 핵심 특징

- **하이브리드 실행 모델**: 에이전틱(Agentic) + 일반 워크플로우 모드 지원
- **다중 LLM 지원**: 로컬(Ollama) + 클라우드(OpenAI, Anthropic 등) 하이브리드
- **품질 검증 시스템**: LangGraph 기반 품질 검증 및 신뢰도 계산
- **모듈형 아키텍처**: 플러그인 방식의 컬렉터, 액션, 어댑터
- **Obsidian 통합**: 마크다운 + YAML 프론트매터 기반 콘텐츠 관리

## 3. 주요 컴포넌트 및 역할

### 3.1 오케스트레이션 레이어

#### 3.1.1 WorkflowEngine
- **역할**: YAML 정의 워크플로우 실행 엔진
- **기능**:
  - 순차 단계 실행
  - 조건부 평가
  - 동적 단계 삽입
  - 배치 처리
  - 폴백 실행
- **위치**: `picko/orchestrator/engine.py`

#### 3.1.2 ActionRegistry
- **역할**: 실행 가능한 액션의 등록 및 관리
- **기능**:
  - 액션 핸들러 등록
  - 액션 실행
  - 액션 메타데이터 관리
- **위치**: `picko/orchestrator/actions.py`

### 3.2 수집 시스템 (Collector System)

#### 3.2.1 BaseCollector
- **역할**: 모든 컬렉터의 추상 기반 클래스
- **기능**:
  - 계정 단위 수집 인터페이스 정의 (`collect(account_id: str)`)
  - 컬렉터 식별 인터페이스 정의 (`name()`)
- **위치**: `picko/collectors/__init__.py`

#### 3.2.2 RSSCollector
- **역할**: RSS 피드에서 콘텐츠 수집
- **기능**:
  - feedparser를 통한 RSS 파싱
  - 콘텐츠 추출 및 정규화
- **위치**: `picko/collectors/rss.py`

#### 3.2.3 PerplexityCollector
- **역할**: Perplexity Task 결과 수집
- **기능**:
  - 파일 시스템 스캔
  - HTML/Markdown 파싱
- **위치**: `picko/collectors/perplexity.py`

### 3.3 품질 시스템 (Quality System)

#### 3.3.1 QualityGraph
- **역할**: LangGraph 기반 품질 검증 상태 머신
- **기능**:
  - 1차 검증 (Primary)
  - 2차 교차 검증 (Cross-Check)
  - 신뢰도 계산
  - 최종 판정
- **위치**: `picko/quality/graph.py`

#### 3.3.2 검증기 (Validators)
- **PrimaryValidator**: 1차 LLM 기반 검증
- **CrossCheckValidator**: 다른 프로바이더로 독립 검증
- **위치**: `picko/quality/validators/`

### 3.4 생성 시스템 (Generator System)

#### 3.4.1 ContentGenerator
- **역할**: 다양한 형식의 콘텐츠 생성
- **기능**:
  - 롱폼 아티클 생성
  - 소셜 미디어 팩 생성
  - 이미지 프롬프트 생성
- **위치**: `picko/generator.py` (스크립트에서 사용)

#### 3.4.2 TemplateRenderer
- **역할**: Jinja2 템플릿 기반 콘텐츠 렌더링
- **기능**:
  - 템플릿 로드
  - 변수 치환
  - 마크다운 생성
- **위치**: `picko/templates.py`

### 3.5 핵심 서비스 (Core Services)

#### 3.5.1 LLMClient
- **역할**: 다중 LLM 프로바이더 통합 클라이언트
- **기능**:
  - 여러 LLM 프로바이더 지원
  - 자동 폴백
  - 스트리밍 지원
- **위치**: `picko/llm_client.py`

#### 3.5.2 EmbeddingManager
- **역할**: 텍스트 임베딩 및 유사도 검색
- **기능**:
  - 로컬/클라우드 임베딩
  - 캐싱
  - 중복 탐지
- **위치**: `picko/embedding.py`

#### 3.5.3 VaultIO
- **역할**: Obsidian Vault 마크다운 I/O
- **기능**:
  - 파일 읽기/쓰기
  - YAML 프론트매터 파싱
  - 디렉토리 관리
- **위치**: `picko/vault_io.py`

## 5. 아키텍처 원칙

### 5.1 분리 원칙 (Separation of Concerns)
- 각 컴포넌트는 명확한 단일 책임을 가짐
- 수집, 검증, 생성, 게시는 완전히 분리
- 의존성 주입을 통한 느슨 결합
