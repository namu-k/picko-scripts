## 개요

### 설계 목표
- 실행 중심의 직관적인 UI 제공
- 파라미터 누락 방지를 단계적 입력 구조화
- 실시간 피드백 기반 UX
- 운영 효율성을 위한 빠른 접근성

### 핵심 원칙
1. **One Task, One Page**: 각 화면은 단일 목적
2. **Progressive Disclosure**: 기본 값 제공, 필요 시만 입력
3. **Live Feedback**: 실행 시 실시간 로그/상태 제공
4. **Pattern Consistency**: 동일 패턴 재사용

## 핵심 사용자 플로우

### 1. 일일 콘텐츠 처리 플로우
```
[ 대시보드 ]
    ↓
[ /run/collect ] → 실행 → [ /status ] (실시간 모니터링)
    ↓
[ /inbox ] → [auto_ready] 선택 → [ Run Pipeline ]
    ↓
[ /status ] → 생성 완료 확인
    ↓
[ /run/generate ] → 실행 → 결과 확인
```

> 참고: MVP v1 빠른 실행 모드는 대시보드 인라인 모달에서 `sources` 선택을 생략하고 전체 소스를 기본으로 사용한다.

### 2. 동영상 제작 플로우
```
[ /video/new ] → 기본 정보 입력
    ↓
[ Source 선택 ] → 기존 글 or 새로운 주제
    ↓
[ Brand Style 설정 ] → 샷 자동 생성
    ↓
[ /video/:id ] → 프롬프트 확인/수정
    ↓
[ 렌더링 요청 ]
```

### 3. 설정 관리 플로우
```
[ /settings ]
    ↓
[ 탭 선택: LLM/Vault/Scoring... ]
    ↓
[ 필드 수정 → 저장 ]
    ↓
[ 즉시 적용 확인 ]
```

## 인터랙션 패턴

### 1. 실행 플로우
1. **선택 → 확인 → 실행**

## 접근성 고려사항

1. **화면 읽기기 지원**
   - 모든 컴포넌트에 적절한 ARIA 레이블
   - 키보드 네비게이션 완벽 지원
   - 포커스 시각적 표시

2. **색상 대비**
   - 텍스트 대비 비율 4.5:1 이상
   - 색상만 의존하지 않는 인디케이터

3. **반응형 디자인**
   - 모바일 우선 접근
   - 터치 가능 영역 최소 48x48px
   - 가로 스크롤 최소화
