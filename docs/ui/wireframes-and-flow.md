## 개요

### 설계 목표
- 실행 중심의 직관적인 UI 제공
- 파라미터 누락 방지를 단계적 입력 구조화
- 실시간 피드백 기반 UX
- 운영 효율성을 위한 빠른 접근성

### 핵심 원칙
1. **One Task, One Page**: 각 화면은 단일 목적
2. **Progressive Disclosure**: 기본 값 제공, 필요 시만 입력
3. **Live Feedback**: 실행 시 실시간 로그/상태 제공
4. **Pattern Consistency**: 동일 패턴 재사용

## 핵심 사용자 플로우

### 1. 일일 콘텐츠 처리 플로우
```
[ 대시보드 ]
    ↓
[ /run/collect ] → 실행 → [ /status ] (실시간 모니터링)
    ↓
[ /inbox ] → [auto_ready] 선택 → [ Run Pipeline ]
    ↓
[ /status ] → 생성 완료 확인
    ↓
[ /run/generate ] → 실행 → 결과 확인
```

### 2. 동영상 제작 플로우
```
[ /video/new ] → 기본 정보 입력
    ↓
[ Source 선택 ] → 기존 글 or 새로운 주제
    ↓
[ Brand Style 설정 ] → 샷 자동 생성
    ↓
[ /video/:id ] → 프롬프트 확인/수정
    ↓
[ 렌더링 요청 ]
```

### 3. 설정 관리 플로우
```
[ /settings ]
    ↓
[ 탭 선택: LLM/Vault/Scoring... ]
    ↓
[ 필드 수정 → 저장 ]
    ↓
[ 즉시 적용 확인 ]
```

#### 3. Intent별 구조 안내

| Intent | 권장 길이 | 샷 수 | 필수 구조 | 용도 |
|--------|----------|-------|----------|------|
| ad | 15-30초 | 3-5개 | CTA 샷 필수 | 전환/다운로드 유도 |
| explainer | 45-120초 | 5-8개 | intro 권장 | 개념 설명/교육 |
| brand | 15-60초 | 3-5개 | (없음) | 브랜드 인지도/분위기 |
| trend | 15-30초 | 3-4개 | (없음) | 트렌드 반응/시의성 |

#### 4. 서비스별 프롬프트 가이드

| 서비스 | 최대 길이 | 지원 비율 | 특징 |
|--------|----------|----------|------|
| Luma | 5초 | 16:9, 9:16, 1:1 | 감성/몽환적, 루프 영상 |
| Runway | 10초 | 16:9, 9:16, 1:1, 4:3 | 고품질 광고 컷, 카메라 워크 |
| Pika | 5초 | 16:9, 9:16, 1:1 | 빠른 실험, 특수 효과 (Pikaffects) |
| Kling | 15초 | 16:9, 9:16 | 장편 설명/튜토리얼 |
| Veo | 8초 | 16:9, 9:16 | 오디오 자동 생성, 프리미엄 |

#### 5. 품질 평가 차원

| 차원 | 가중치 | 평가 기준 |
|------|--------|----------|
| Prompt Quality | 25% | 구체성, 시각 묘사, 추상 표현 배제 |
| Structure | 20% | Intent별 필수 구조 준수 |
| Brand Alignment | 15% | 톤/비율 일관성 |
| Platform Fit | 20% | 길이/비율 제약 준수 |
| Actionability | 20% | 서비스 파라미터 완전성 |

## 인터랙션 패턴

### 1. 실행 플로우
1. **선택 → 확인 → 실행**
2. **즉시 피드백** (로스트 포커스)
3. **진행 표시기** 시각적 피드백
4. **실시간 로그** 실행 중 업데이트

### 2. 설정 패턴
1. **Tab으로 그룹 분리**
2. **저장 전 변경점 요약**
3. **변경 시 즉시 적용**
4. **Rollback 옵션**

### 3. 목록 인터랙션
1. **Bulk 선택 (Select All)**
2. **편집/삭제 inline**
3. **필터링/정렬 기본 제공**
4. **상세 화면 진입**

### 4. 알림 시스템
1. **Toast 알림** (자동 사라짐)
2. **모달 확인** (중요 액션)
3. **로그 스트림** (실시간)
4. **Badge 카운터** (미확인 항목)

## 접근성 고려사항

1. **화면 읽기기 지원**
   - 모든 컴포넌트에 적절한 ARIA 레이블
   - 키보드 네비게이션 완벽 지원
   - 포커스 시각적 표시

2. **색상 대비**
   - 텍스트 대비 비율 4.5:1 이상
   - 색상만 의존하지 않는 인디케이터

3. **반응형 디자인**
   - 모바일 우선 접근
   - 터치 가능 영역 최소 48x48px
   - 가로 스크롤 최소화

## 기술 요구사항

1. **UI 라이브러리**: React + Tailwind CSS
2. **상태 관리**: Redux Toolkit 또는 Zustand
3. **실시간 통신**: Server-Sent Events (SSE)
4. **라우팅**: React Router v6
5. **테스트**: React Testing Library + Jest
