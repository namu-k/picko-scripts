# Picko Contents ExEvGen(탐색평가생성) 프로젝트 소개

## ExEvGen이란?

**ExEvGen**는 RSS 피드와 웹 소스에서 콘텐츠를 자동으로 수집·가공하고, 여러 LLM을 활용해 **롱폼 글**, **소셜 미디어 포스트**, **이미지 프롬프트** 등을 생성하는 **콘텐츠 파이프라인 자동화 시스템**입니다. 생성된 콘텐츠는 **Obsidian Vault** 구조의 마크다운 노트로 관리됩니다.

```
 RSS 수집 → 콘텐츠 추출 → NLP 처리 → 점수 산정 → 다이제스트 생성 → 승인 → Longform 생성 → 검증 완료
```

---

## 핵심 기능

### 1. 일일 수집 파이프라인 (Daily Collector)

- **RSS 피드 수집**: `config/sources.yml`에 정의된 소스에서 기사·포스트를 가져옵니다.
- **전처리**: 중복 제거, 본문 추출, 요약·태깅·임베딩 등 NLP 처리를 수행합니다.
- **점수 계산**: 참신도(novelty), 관련도(relevance), 품질(quality) 가중치로 점수를 매겨 우선순위를 정합니다.
- **다이제스트 생성**: 날짜별 다이제스트(`Inbox/Inputs/_digests/`)를 만들고, 각 항목을 개별 입력 노트(`Inbox/Inputs/`)로 저장합니다.

### 2. 콘텐츠 생성 (Generate Content)

- 다이제스트에서 **승인된 항목**(체크된 항목, `writing_status: auto_ready`)만 선별합니다.
- **롱폼 글**, **소셜 팩**(트위터·링크드인 등), **이미지용 프롬프트**를 Jinja2 템플릿과 LLM으로 생성합니다.
- 결과는 Vault의 `Content/Longform`, `Content/Packs`, `Assets/Images/_prompts` 등에 마크다운으로 저장됩니다.

### 3. 품질·운영 도구

- **validate_output**: 생성된 콘텐츠의 프론트매터, 섹션, 위키링크 등을 검증합니다.
- **health_check**: API 키, Vault 접근, RSS 소스, 디스크 공간 등 시스템 상태를 점검합니다.
- **archive_manager**, **retry_failed**, **publish_log** 등으로 아카이브, 재시도, 발행 로그를 관리합니다.

### 4. 워크플로우 자동화 (v0.4.0+)

- **run_workflow**: YAML 기반 워크플로우 실행 (`config/workflows/`)
- **Orchestration Layer**: VaultAdapter, WorkflowEngine, ActionRegistry
- **Publisher Module**: Twitter/LinkedIn 발행 지원

### 5. Video 기획 파이프라인 (v0.5.0+)

외부 AI 동영상 서비스(Runway, Pika, Kling, Luma, Veo 등)를 활용해 영상 기획서를 생성합니다:
- **3축 유형 결정**: 소스(source) × 의도(intent) × 주간 맥락(week-of)의 조합으로 영상 유형 결정
- **CLI**: `python -m picko video --intent ad --content <id> --week-of <date>`
- **출력**: `Content/Video/` 경로에 VideoPlan JSON/Markdown 저장
---

## 설계 특징: 작업별 LLM 분리

- **요약·태깅** (`summary_llm`): 로컬 LLM(Ollama 등)을 우선 사용해 비용을 줄이고, 실패 시 클라우드로 폴백합니다.
- **임베딩** (`embedding`): 로컬(sentence-transformers) 또는 Ollama 임베딩 모델로 벡터를 만들고, 캐시로 재사용합니다.
- **글쓰기** (`writer_llm`): 품질이 중요한 본문·소셜 문구 생성에는 OpenAI, Anthropic, OpenRouter, Relay 등 클라우드 LLM을 사용합니다.

설정은 `config/config.yml`과 계정별 프로필(`config/accounts/`)에서 YAML로 관리하며, API 키는 `.env`에서 로드합니다.

---

## 콘텐츠 흐름 요약

1. **수집** → RSS/소스 → 전처리·NLP·점수 → `Inbox/Inputs/` + 일별 다이제스트
2. **선별** → 다이제스트에서 항목 검토 후 `auto_ready` 등으로 승인
3. **생성** → 승인된 항목만 롱폼·팩·이미지 프롬프트 생성 → `Content/`, `Assets/`
4. **Video 기획** → 3축 유형 결정으로 VideoPlan 생성 → `Content/Video/`
5. **검증·발행** → validate_output·health_check·publish_log로 품질 및 배포 관리

이 구조로 **수집 → 큐레이션 → 생성 → 기획 → 검증**까지 한 번에 자동화할 수 있습니다.
