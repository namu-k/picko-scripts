# Feature Specification: Pipeline UX (프롬프트·초안·알림·다음 명령)

**Feature Branch**: `002-pipeline-ux`
**Created**: 2026-02-18
**Status**: Draft
**Input**: 파이프라인 사용성 개선 — 프롬프트 품질, 에이전트-운영자 소통(소통 방법 + 폴백). 초안 선택·알림·다음 명령 제안은 소통이 필요한 지점의 예시.

## User Scenarios & Testing *(mandatory)*

### 영역 1: 프롬프트 개선 (롱폼, 팩, 이미지) — Priority: P1

**User Story**: 운영자는 계정·채널 변수(타깃 독자, 톤, 형식)가 반영된 프롬프트로 롱폼/팩/이미지 생성 품질을 높일 수 있어야 한다.

**Why this priority**: 품질 개선이 없으면 이후 초안 선택·알림의 가치가 줄어든다.

**Independent Test**: 계정 프로필·스타일이 프롬프트에 주입되는지 확인. 생성 결과에 톤·형식이 반영되는지 샘플 검증.

**Acceptance Scenarios**:

1. **Given** 계정 정체성(one_liner, target_audience, tone_voice)이 설정됨, **When** 롱폼/팩/이미지 프롬프트가 로드됨, **Then** 해당 변수가 Jinja2 템플릿에 치환되어 LLM에 전달된다.
2. **Given** 채널별 형식 지시(예: 트위터 글자 수, 링크드인 톤)가 설정됨, **When** 팩 프롬프트가 사용됨, **Then** 채널 변수가 반영된 지시문이 포함된다.
3. **Given** 스타일 프로필(참조 URL 기반)이 있음, **When** 글쓰기 프롬프트가 조합됨, **Then** 스타일 레이어가 base 프롬프트 위에 적용된다.

---

### 영역 2: 에이전트-운영자 소통 — 예: 2~3개 초안 생성 후 사람이 선택 (Priority: P1)

**원칙**: 에이전트와 운영자 간 소통이 필요한 지점마다, 적절한 소통 방법과 폴백 소통 방법이 마련되어야 한다.

**User Story (예시)**: 운영자는 롱폼(또는 팩)에 대해 2~3개 초안이 생성되고, 선택한 초안만 “승인”으로 표시되어 다운스트림에 사용되도록 할 수 있어야 한다. 선택을 전달하는 1차 방법(예: 파일/프론트매터)과, 그 불가 시 폴백(예: 터미널 안내, 기본 초안 사용)이 정의되어야 한다.

**Why this priority**: 소통 지점마다 1차·폴백이 있어야 파이프라인이 안정적으로 동작한다. (예시로서 품질·선택권 확보.)

**Independent Test**: generate_content 실행 시 N개 초안이 생성·저장되고, “선택된 초안” 필드/파일로 표시되며, 해당 초안만 후속 처리에 사용되는지 확인.

**Acceptance Scenarios**:

1. **Given** 설정에서 초안 개수 N(2 또는 3)이 지정됨, **When** 롱폼(또는 팩) 생성 요청, **Then** N개의 초안이 동일 입력에 대해 생성되어 저장된다.
2. **Given** N개 초안이 저장됨, **When** 운영자가 하나를 “선택된 초안”으로 표시(파일/프론트매터 또는 별도 표시), **Then** 다운스트림(이미지 생성, 발행 로그 등)은 선택된 초안만 참조한다.
3. **Given** 초안이 아직 선택되지 않음, **When** 이미지 생성 또는 발행 스크립트 실행, **Then** “선택된 초안이 없음” 등 명확한 안내가 나오거나 해당 항목은 스킵된다.

---

### 영역 3: 알림(텔레그램 등) 발송 + “응답 완료” 신호 — Priority: P2

**User Story**: 운영자는 일 수집 직후 또는 생성 전/후 등 정해진 시점에 알림을 받고, 파이프라인/봇이 “응답 완료” 신호를 보낼 수 있어야 한다. 알림 수단(1차)과 폴백(알림 미설정·실패 시 로그/완료 파일 등)이 정의되어야 한다.

**Why this priority**: 소통 지점의 1차·폴백 보장. 수동 확인 부담 감소, 외부 스케줄러·봇 연동.

**Independent Test**: daily_collector 또는 generate_content 종료 시 알림이 발송되는지, “완료” 신호(파일 터치, 웹훅, 봇 메시지 등)가 정의대로 동작하는지 확인.

**Acceptance Scenarios**:

1. **Given** 알림 수단(텔레그램 봇 등)이 설정됨, **When** daily_collector가 정상 종료함, **Then** 지정된 시점(일 수집 직후 등)에 알림이 발송된다.
2. **Given** “응답 완료” 신호가 파일 터치로 정의됨, **When** 스크립트가 성공적으로 끝남, **Then** 지정 경로에 완료 마커 파일이 생성된다.
3. **Given** 알림 수단이 설정되지 않음, **When** 스크립트 실행, **Then** 알림 단계는 스킵되고 나머지 파이프라인은 정상 동작한다.

---

### 영역 4: 다음 명령 제안 / 필요 시 자동 진행 — Priority: P2

**User Story**: 운영자는 daily_collector 또는 generate_content 종료 시 “다음에 실행할 명령” 문구를 터미널(또는 로그)에서 확인하고, 추가 개입이 불필요한 경우 “완료”만 표시할 수 있어야 한다. (선택) 자동 진행 조건이 있으면 그에 따라 다음 단계가 실행된다.

**Why this priority**: 워크플로 명확화. 다음에 무엇을 해야 할지 한눈에 파악.

**Independent Test**: 스크립트 종료 시 출력/로그에 다음 명령 문구가 포함되는지, “추가 개입 불필요” 시 완료 메시지만 나오는지 확인.

**Acceptance Scenarios**:

1. **Given** daily_collector가 성공적으로 완료됨, **When** 종료 시점, **Then** 터미널(또는 로그)에 “다음 권장 명령: python -m scripts.generate_content --date YYYY-MM-DD” 등이 출력된다.
2. **Given** generate_content가 성공적으로 완료됨, **When** 종료 시점, **Then** “다음 권장 명령” 또는 “추가 개입 불필요. 완료.” 등이 상황에 맞게 출력된다.
3. **Given** (선택) 자동 진행 조건이 설정됨(예: --auto-proceed), **When** 조건 충족, **Then** 다음 단계 스크립트가 자동 실행되거나, 그렇지 않으면 수동 명령만 제안된다.

---

## Edge Cases

- 프롬프트 변수(계정·채널)가 누락된 경우: 기본값 사용 또는 명확한 경고 로그 후 계속 진행.
- 초안 개수 N이 1인 경우: 기존 단일 생성 동작 유지(하위 호환).
- 알림 발송 실패 시: 파이프라인 자체는 성공으로 처리하고, 알림 실패만 로그/재시도 정책으로 처리.
- “다음 명령” 제안이 잘못된 날짜/옵션을 포함할 수 있는 경우: 실제 실행된 날짜·옵션을 반영한 문구 생성.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 롱폼/팩/이미지 프롬프트에 계정 정체성·채널 변수·스타일 프로필이 반영된다.
- **FR-002**: 에이전트-운영자 소통이 필요한 지점(예: 초안 선택)마다 1차 소통 방법과 폴백 소통 방법이 정의된다. (예: 설정으로 롱폼/팩당 초안 N개 저장, 선택 표시 방식 및 선택 없을 때 폴백.)
- **FR-003**: “선택된 초안” 표시 방식과 다운스트림 사용 방식이 정의되고, 선택 불가 시 폴백(안내 또는 기본값)이 있다. (예시)
- **FR-004**: 알림 발송 시점·수단(1차)과, 미설정/실패 시 폴백(로그, 완료 파일 등)을 설정할 수 있다.
- **FR-005**: “응답 완료” 신호의 1차 수단과 폴백(예: 파일 터치 실패 시 로그만)이 정의된다.
- **FR-006**: 종료 시 다음 권장 명령 문구가 출력(1차)되며, 필요 시 로그 등 폴백 경로에도 기록할 수 있다.
- **FR-007**: 추가 개입이 불필요한 경우 “완료” 등 간단 메시지로 표시할 수 있다.

### Key Entities

- **프롬프트**: config/prompts/ 기반, 계정·채널·스타일 변수 치환.
- **에이전트-운영자 소통**: 소통이 필요한 지점마다 1차 소통 방법 + 폴백 소통 방법 정의. (예: 초안 선택, 알림, 다음 명령 제안.)
- **초안(예시)**: 동일 입력에 대한 N개 생성 결과, 선택 표시 및 폴백. (저장 형식·선택 플래그는 구현 시 정의)
- **알림 모듈**: 발송 시점·1차 수단·폴백(로그/파일 등).

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 계정/채널 변수가 적용된 프롬프트로 롱폼·팩·이미지가 생성된다.
- **SC-002**: N(2 또는 3)개 초안 생성 및 선택된 초안만 다운스트림에 사용되는 플로우가 동작한다.
- **SC-003**: 설정된 시점에 알림이 발송되고(설정 시), 응답 완료 신호가 정의대로 동작한다.
- **SC-004**: 스크립트 종료 시 다음 명령 제안 또는 완료 메시지가 출력된다.
- **SC-005**: 기존 단일 초안·알림 없는 플로우는 설정으로 유지 가능(하위 호환).
