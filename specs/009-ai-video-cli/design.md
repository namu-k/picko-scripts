## AI 동영상 제작 서비스 활용 전략 (브랜치 009)

이 문서는 Picko 프로젝트가 **직접 동영상을 렌렌더링하지 않고**, 상용 AI 동영상 제작/편집 서비스를 최대한 잘 활용하기 위한 전략과 통합 포인트를 정리한다.

- **Picko의 역할**: 원본 콘텐츠 수집·요약·구조화 → 동영상 기획/스크립트/샷 리스트/브랜드 가이드 생성 → 외부 서비스에 넘길 표준 포맷 정의
- **외부 서비스의 역할**: 비디오 렌더링, 타임라인 편집, 아바타·음성 합성, 이펙트/모션 적용

---

## 1. 주요 서비스 분류

### 1.1 창의/시네마틱 동영상 생성형

이 카테고리는 **텍스트/이미지/기존 클립을 입력으로 받아 시네마틱 영상을 직접 합성하는 모델들**을 포함한다.  
브랜드/캠페인용 숏폼, 시각적 임팩트가 큰 B-roll, 광고 컷 등에 주로 사용한다.

#### Runway (Gen-4.x/4.5)

- **포지션**: 시네마틱 숏폼, 광고/캠페인형 영상의 “실무 표준” 후보
- **강점**
  - 텍스트/이미지 → 비디오, 이미지/비디오 리스타일 등 다양한 모드
  - 카메라 워크·구도·조명 등 **디렉션 프롬프트 반영력**이 가장 안정적인 축
  - 4K까지 커버 가능한 모델 라인업, 웹 앱 + API 모두 지원
- **제약**
  - 크레딧 기반 과금, 고품질 모드는 비용·속도 모두 부담 → “선택된 샷”에 쓰는 것이 현실적
- **Picko에서의 역할**
  - 브랜드 인트로/아웃로, 메인 캠페인 컷처럼 **비싸도 임팩트가 큰 샷**을 담당
  - `VideoPlan`에서 Runway 전용으로 카메라/조명/분위기 필드를 더 정교하게 기획

#### Pika (Pika 2.x/2.5)

- **포지션**: SNS/숏폼 중심의 빠른 반복 생산기
- **강점**
  - 3–10초 단위 클립을 **수 초~수십 초 안에 생성** → 반복 실험에 최적
  - 소셜용 효과(Pikaffects, Pikaswaps 등), 16:9/9:16/1:1 등 다양한 비율
- **제약**
  - Runway/Veo 대비 “완전 시네마틱 광고”급 퀄리티는 한 단계 아래
  - 내러티브 긴 영상보다는 **짧은 메시지 중심**에 적합
- **Picko에서의 역할**
  - 데일리/위클리 digest에서 **짧은 인사이트 숏폼**을 양산하는 라인
  - `VideoPlan`에서 Pika 타겟 샷은 훅 한 줄 + 간단한 배경 프롬프트 + 9:16 전제를 기본값으로 설계

#### Kling 3.0

- **포지션**: **길이·비용 측면에서 가장 실용적인 장편 후보**
- **강점**
  - 단일 클립 15초, 연장 기능으로 최대 3분까지 생성 가능 → 타 서비스 대비 가장 긴 축
  - 1080p 기본, 상위 플랜에서 네이티브 4K 지원
  - 초당 단가가 저렴해 **긴 러닝타임 + 반복 시도**에 적합
- **제약**
  - 인터페이스/에코시스템이 Runway 만큼 정제되어 있진 않음
- **Picko에서의 역할**
  - 1–2분짜리 **개념 설명/튜토리얼형 영상**을 한 번에 합성하고 싶을 때 우선 고려
  - `VideoPlan`에서 Kling 전용으로 도입–본론–결론 구조와 씬 연결 힌트를 넣는 것을 상정

#### Luma Dream Machine

- **포지션**: 속도 좋은 시네마틱 B-roll/루프 생성기
- **강점**
  - 5초 단위 시네마틱 클립을 아주 빠르게 생성
  - 비디오 전용 트랜스포머로 학습되어 물리/카메라 무빙/일관성이 우수
- **제약**
  - 길이가 짧아 내러티브형 장편보다는 **짧은 시각 효과용**에 적합
- **Picko에서의 역할**
  - 설명/아바타 영상의 배경으로 쓸 **B-roll, 루프형 시네마틱 클립**을 뽑는 데 사용
  - `VideoPlan`에서 “배경 전용 샷”을 Luma 대상으로 태깅할 수 있음

#### Google Veo 3.x

- **포지션**: 품질·안정성이 최고 수준인 프리미엄 모델
- **강점**
  - 여러 비교에서 영상 품질 점수 최상위, 네이티브 오디오(대사/효과음/배경음)까지 동시 생성
  - 720p/1080p/4K 업스케일, 9:16 포함 다양한 비율, 씬 확장/보간 등 편집형 기능 일부 내장
- **제약**
  - 초당 비용이 높고, 1회 생성 길이가 8초 세그먼트로 제한
- **Picko에서의 역할**
  - “브랜드 메인 영상”처럼 **정말 중요한 소수 씬**에만 선택적으로 사용
  - `VideoPlan`에서 Veo 대상으로는 오디오 분위기/사운드 톤까지 포함한 디테일을 정의

#### OpenAI Sora 2

- **포지션**: 비주얼 연구·데모 기준점, 실무 대량 생산에는 제약 다수
- **강점**
  - 포토리얼리즘, 복잡한 물리·장면 구성 등에서 “쇼케이스” 기준 최상급
- **제약**
  - 짧은 길이(수 초~수십 초), 접근성/가격/속도 제약, 캐릭터 일관성·편집 기능 한계
  - 현재는 API/지역 제한으로 **프로덕션 워크플로의 메인 축으로 삼기 어렵다**
- **Picko에서의 역할**
  - 상용 파이프라인의 기본 옵션이 아니라, **아이디어/컨셉 시각화용 실험 채널**로만 고려

## 2. Picko가 제공할 표준 영상 기획 포맷 초안

Picko는 상용 서비스마다 포맷을 따로 만들기보다, 내부적으로 **표준 VideoPlan 포맷**을 만들고, 서비스별 어댑터가 이를 변환하는 구조를 목표로 한다.

### 2.1 VideoPlan 스키마 (초안)

```json
{
  "id": "video_2026-03-05_001",
  "account": "socialbuilders",
  "source": {
    "type": "longform",
    "id": "lf_2026-03-01_123"
  },
  "goal": "예비창업자에게 MVP 개념을 직관적으로 설명",
  "target_services": ["runway", "pika", "heygen"],
  "platforms": ["instagram_reel", "youtube_short"],
  "duration_sec": 45,
  "brand_style": {
    "theme": "socialbuilders",
    "tone": "친근하지만 전문적",
    "colors": {},
    "fonts": {}
  },
  "shots": [
    {
      "index": 1,
      "type": "intro",
      "duration_sec": 5,
      "script": "MVP는 최소 기능 제품을 의미해요. 완벽함보다 학습이 더 중요합니다.",
      "caption": "MVP, 왜 필요할까?",
      "background_prompt": "modern tech gradient, startup founder, korean office",
      "notes": {
        "runway": "시네마틱 카메라 인, 로고를 살짝 노출",
        "pika": "9:16 세로, 텍스트 강조",
        "heygen": "아바타가 카메라를 보며 설명"
      }
    }
  ]
}
```

- `brand_style.colors`, `brand_style.fonts`는 기존 `config/layouts/` + `config/accounts/` 정보를 재사용해 채운다.
- `shots[*].notes` 안에 서비스별 세부 힌트를 두되, **핵심 정보(대사/자막/길이)는 공통 필드**로 유지한다.

## 3. 서비스별 연동 전략 (요약)

### 3.1 창의/시네마틱 동영상 생성형 (Runway / Pika / Kling / Luma / Veo / Sora)

1.1에서 분류한 서비스들이 공통으로 받을 입력과 Picko가 담당하는 기획 작업을 정리한다.

- **입력**:
  - `VideoPlan.shots[*].script` → 장면 설명/캡션 텍스트
  - `VideoPlan.shots[*].background_prompt` → 텍스트 프롬프트
  - `VideoPlan.shots[*].notes[service]` → 서비스별 세부 디렉션 힌트
  - aspect ratio, 길이, 스타일은 `brand_style` + 계정/플랫폼 기본값에서 가져오기
- **Picko가 해줄 일**:
  - digest/longform/계정 설정에서 핵심 메시지를 뽑아 **샷 리스트로 구조화**
  - 샷별 길이·강조 포인트·카메라 무드·배경 프롬프트를 자동 제안
  - `target_services`에 명시된 서비스 특성에 맞게 `shots[*].notes`를 채우기
    - Runway → 카메라 워크·조명·시네마틱 방향
    - Pika → 9:16, 짧은 훅 중심, 효과 제안
    - Kling → 도입-본론-결론 씬 연결 힌트
    - Luma → 배경 전용 B-roll·루프 용도 명시
    - Veo → 오디오 분위기·사운드 톤까지 포함
    - Sora → 실험/컨셉 시각화 용도로만 표기

## 4. 향후 구현 방향 (009 브랜치 기준 제안)

1. **VideoPlan 데이터 모델 정의**
   - 위치 후보: `picko/video_plan.py` (dataclass 형태)
   - 필수 필드: `id, account, source, goal, platforms, target_services, brand_style, shots[*]`
2. **생성 파이프라인 연결**
   - 입력: `Inbox/Inputs/` 또는 longform/pack 결과물
   - 출력: Vault 내 `Inbox/Video/` 이하에 VideoPlan을 마크다운/JSON으로 기록
3. **서비스별 어댑터 스텁**
   - 예: `scripts/export_video_runway.py`, `scripts/export_video_heygen.py`
   - 초기에는 파일로만 저장하고, 이후 실제 API 연동 가능하면 확장

이 문서는 009 브랜치에서 상용 동영상 제작 서비스 활용을 위한 **기준 설계 문서**로 사용하며, 실제 구현이 진행되면 각 어댑터/워크플로에 대한 세부 문서를 추가로 분리한다.
