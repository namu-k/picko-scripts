# Picko - AI 콘텐츠 파이프라인

**수집부터 발행까지, 콘텐츠 큐레이션을 완전 자동화하세요.**

---

## Picko가 해결하는 문제

### 콘텐츠 운영의 3가지 병목

| 문제 | 기존 방식 | Picko 방식 |
|------|----------|-----------|
| **정보 과부하** | 수십 개 사이트를 매일 방문 | RSS 자동 수집 + AI 필터링 |
| **시간 소모** | 아티클 1편 작성에 2-4시간 | 10분 내외로 초안 생성 |
| **일관성 부재** | 톤앤매너가 매번 달라짐 | 페르소나 기반 일관된 스타일 |

**결과**: 콘텐츠 제작 시간 80% 단축, 일일 1시간 투자로 다채널 운영 가능

---

## 핵심 가치

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   📥 수집      →  🧠 분석     →  ✍️ 생성     →  📤 발행         │
│   (자동화)        (AI)          (AI)          (검토 후)         │
│                                                                 │
│   RSS 피드       요약/태깅     블로그 포스트   Obsidian         │
│   뉴스 사이트    점수 매기기   소셜 미디어     마크다운         │
│   웹 소스        품질 필터     이미지 프롬프트                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1. 스마트 필터링

AI가 수집된 콘텐츠를 평가합니다:
- **참신도** (30%): 기존 콘텐츠와의 차별성
- **관련도** (40%): 설정한 관심사와의 일치도
- **품질** (30%): 본문 길이, 출처 신뢰도

→ 점수 0.85 이상만 자동 승인, 0.4 미만은 자동 제외

### 2. 멀티 포맷 생성

하나의 소스에서 여러 형식을 자동 생성:

| 포맷 | 용도 | 특징 |
|------|------|------|
| **Longform** | 블로그, 미디엄 | 2,000~5,000자 심층 아티클 |
| **Twitter** | 트위터/스레드 | 280자 내외, 캐주얼 톤 |
| **LinkedIn** | 링크드인 | 1,500자, 전문적 톤 |
| **Newsletter** | 이메일 뉴스레터 | 구조화된 섹션, 정보 중심 |
| **Image Prompt** | AI 이미지 | 썸네일/일러스트용 프롬프트 |

### 3. 페르소나 기반 일관성

계정별로 정의된 페르소나가 모든 콘텐츠에 반영됩니다:
- 타겟 오디언스 정의
- 말투와 톤 (전문적/친근/정보성)
- 핵심 가치 (Pillars)
- 주간 콘텐츠 전략

## 기술적 차별점

### 하이브리드 LLM 아키텍처

비용 최적화를 위해 작업별로 다른 AI를 사용합니다:

| 작업 | 사용 AI | 이유 | 비용 |
|------|---------|------|------|
| 요약/태깅 | **로컬 LLM** (Ollama) | 대량 처리, 복잡도 낮음 | **무료** |
| 임베딩 | **로컬 모델** (BGE-M3) | 유사도 계산용 벡터화 | **무료** |
| 글쓰기 | **클라우드 LLM** (GPT-4o) | 창의적 퀄리티 필요 | 유료 |

**일일 운영비**: 하루 10개 콘텐츠 기준 **약 $0.10** (로컬 LLM 활용 시)

### Obsidian 네이티브

모든 콘텐츠가 Obsidian Vault에 마크다운으로 저장됩니다:
- 버전 관리 용이 (Git 연동 가능)
- 위키링크로 지식 그래프 구축
- 플러그인 생태계 활용
- 로컬 데이터 완전 소유

### 유연한 확장성

```yaml
# 새 소스 추가: config/sources.yml
- id: "my_blog"
  type: "rss"
  url: "https://myblog.com/feed"
  category: "custom"
  enabled: true

# 새 계정 추가: config/accounts/new_channel.yml
account:
  id: "new_channel"
  interests: ["테크", "비즈니스"]
```

## 사용 시나리오

### 1. 개인 브랜딩
> "매일 테크 뉴스를 큐레이션해서 트위터와 링크드인에 올리고 싶어요"

→ Picko가 수집 → 필터링 → 승인된 항목만 자동으로 멀티플랫폼 포스트 생성

### 2. 스타트업 콘텐츠 마케팅
> "업계 트렌드를 정리한 뉴스레터를 주 1회 발행하고 싶어요"

→ 주간 수집 → 점수 상위 항목 선별 → 뉴스레터 포맷으로 자동 생성

### 3. 미디어 운영
> "여러 출처의 AI 뉴스를 종합해서 심층 아티클을 쓰고 싶어요"

→ AI 카테고리 소스 집중 수집 → 주제 탐색 → 롱폼 아티클 생성

### 4. 연구/학습
> "관심 분야의 최신 논문과 아티클을 정리해두고 싶어요"

→ RSS 수집 → 요약/태깅 → Obsidian으로 개인 지식 베이스 구축

## 시작하기

### 최소 요구사항
- Python 3.13+
- OpenAI API 키 (또는 Anthropic/OpenRouter)
- Obsidian (선택사항)

### 10분 설치

```bash
# 1. 클론
git clone https://github.com/namu-k/picko-scripts.git
cd picko-scripts

# 2. 가상환경
python -m venv .venv && .venv\Scripts\activate

# 3. 설치
pip install -r requirements.txt

# 4. API 키 설정
echo OPENAI_API_KEY=sk-xxx > .env

# 5. 첫 실행
python -m scripts.daily_collector --dry-run
```
